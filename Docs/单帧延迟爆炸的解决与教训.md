graphicsRenderWorldGridEnabled graphicsRenderPreySignalsEnabled graphicsRenderPreyDamageOverlaysEnabled graphicsRenderPreyAttachmentMarksEnabled graphicsRenderPreyAttachmentHaloEnabled graphicsRenderPredationLinksEnabled graphicsRenderFormationGlowEnabled 也就是说，打开“极简渲染模式”时，实际执行的是把上面这些都设成 false，实现逻辑在 tuning-panel.js#L1006 和 tuning-panel.js#L1846。 结合渲染实现，具体少画了哪些东西 世界网格被关掉了 对应 sceneRender.js#L975 的 drawHud()，里面本来有：顶部能量条面板 24 段能量条分段填充 成长条/压力条 阶段 pips 底部中心状态圈 死亡/胜利全屏覆盖 这层视觉信息多，但对“看清场上单位”不是核心，所以主观画质损失没有想象中大。 prey 的“信号圈”被关掉了 开关接在 sceneRender.js#L715。 这会让 drawPrey() / renderPreySprites() 不再画：guard pulse 圈 weakspot 标记 objective 双环 这类东西单看很小，但它们是“每只 prey 都可能额外多几笔描边/圆形”的典型高频成本。 prey 的“受伤吞食叠层”被关掉了 开关接在 sceneRender.js#L717。 这会停掉 prey 身上这些额外图层：carve 阴影扩层 wound 受伤层 devour glow exposed 弱点暴露层 这一项我很怀疑是“又不显眼又很吃”的大头之一。原因是它不是让你多看到一个完全新物体，而是在每只 prey 主体上再叠 1 到 4 层额外 shape。视觉上只是“更肉、更亮、更惨”，但底层是实打实多画很多路径。 prey 的“咬点标记”被关掉了 开关接在 sceneRender.js#L719。 这部分最隐蔽，但也最危险。每个 attachment 可能会额外画：一个阴影圆 一个 hook/grind/feed 小图形 也就是每个附着点至少两笔，而 prey 附着一多，这部分会呈倍数增长。它主观上只是 prey 表面有点“被咬住”的质感，不会像关掉主体那样显眼，但底层非常可能是 batchFillPath 的重灾区。 prey 的“附着中心环”被关掉了 开关接在 sceneRender.js#L721。 这一项每只有 attachment 的 prey 都会多一圈描边。单个不贵，但数量一多会持续堆。 predation links 整层被关掉了 入口在 sceneRender.js#L301，实际绘制在 sceneRender.js#L823。 这层会为每个 attachment 额外画： 两段线 最多两个圆点 它在画面上其实很像“细节辅料”，没有主体轮廓那么显眼，但 attachment 多的时候，线段和圆点数量会暴涨。 所以这次不是“砍主体”，而是精准砍“装饰性、叠层性、附着性、描边性”的高频路径。 为什么它会“不明显但猛吃延迟” 我认为最像元凶的是这三类： prey overlays 也就是 signals + damage overlays + attachment marks + attachment halo 因为它们都挂在每只 prey 身上，属于“主体之外的重复叠加”。 视觉上你只觉得 prey 更生动了；底层却是每只多画好几层 shape / circle / stroke。 predation links 这是典型的“细线条很多、存在感弱、数量爆炸”。 尤其在大量 attachment 同时存在时，它会按 attachment 数量线性甚至体感超线性变重。 formation glow 这项看起来像“主角更有氛围”，但里面其实有很多动态圆、动态描边、动态小核，尤其每帧都在抖、都在变 alpha/size。 它对可玩性帮助不大，但每帧都算、每帧都画。 按“隐蔽但昂贵”的嫌疑排序 如果让我按“看起来不明显，但最可能贡献大量 batchFillPath”排序，我会先怀疑： graphicsRenderPreyDamageOverlaysEnabled graphicsRenderPreyAttachmentMarksEnabled graphicsRenderPredationLinksEnabled graphicsRenderFormationGlowEnabled graphicsRenderWorldGridEnabled graphicsRenderHudEnabled HUD 排后面，不是说它不贵，而是它“很显眼”；你一关就能明显看出来。你现在反馈“画面并没有降级太多”，反而说明真正伤性能的更可能是 prey 身上那堆不显眼叠层和 attachment 相关图元。 一句话结论 这次极简模式砍掉的，不是“你最看得见的主体”，而是“你不太会注意、但每个单位都在重复叠很多次的小装饰路径”。 所以主观上画面还完整，客观上 batchFillPath 很可能一下少掉一大截。 如果你愿意，我下一步可以直接继续做一个更狠、更适合定位的二分版预设： Prey Overlays Only Off Predation Links Only Off Formation Glow Only Off 这样你下一次抓图，基本就能直接锁定第一嫌疑人。
