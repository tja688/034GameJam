# Phaser 3 音频系统落地方案与调试工作流

## 1. 本次落地范围

本项目已落地统一音频体系，覆盖：

- 统一事件层：业务代码只触发 `playAudioEvent(eventId, meta)`
- 统一管理层：`CoreAudioManager`
- 开发态调试面板：`CoreAudioDebugPanel`（`F8` 切换）
- 配置持久化：`audio-profile.json`
- 音频库索引：`audio-library.manifest.json`
- 场景接入混入：`SceneAudioMixin`

## 2. 关键文件

- `src/audioEventRegistry.js`
  - 维护音频事件注册表（事件名、分组、模块锚点、默认参数）
- `src/audioManager.js`
  - 事件播放、冷却、并发上限、总线音量、资源懒加载、统计与差异导出
- `src/audioDebugPanel.js`
  - 事件浏览/过滤、参数编辑、素材浏览、A/B 试听、运行态应用、保存
- `src/sceneAudio.js`
  - 场景统一入口：初始化管理器、提供 `playAudioEvent` API、挂接调试面板
- `audio-profile.json`
  - 持久化音频配置（总线、已应用绑定、运行态覆盖）
- `audio-library.manifest.json`
  - 音频素材清单（分组、标签、文件路径）
- `tools/generate_audio_manifest.py`
  - 扫描音频目录并生成 `audio-library.manifest.json`

## 3. 配置分层（优先级从低到高）

1. `audioEventRegistry` 默认值（`defaults`）
2. `audio-profile.json` 中 `appliedEventBindings`（点击 Apply + Save 后）
3. `audio-profile.json` 中 `runtimeOverrides`（调试态临时覆盖）
4. 调用 `playEvent` 时的 `overridePatch`（一次性覆盖）

## 4. 运行流程

1. 场景 `create()` 调用 `initAudioSystem()`
2. `CoreAudioManager` 启动时读取：
   - `audio-library.manifest.json`
   - `audio-profile.json`
3. 业务侧通过 `playAudioEvent` 触发事件
4. 管理器执行：
   - 事件配置求值
   - 冷却/并发保护
   - 从素材池选素材
   - 懒加载素材并播放
   - 更新调试统计

## 5. 调试面板工作流（推荐）

1. 启动开发服务器：`./start-dev.ps1`（或 `start-dev.bat`）
2. 进游戏后按 `F8` 打开 Audio Debug
3. 在 Events 区选中目标事件（可搜索/按分组过滤）
4. 在 Library 区试听素材、设置 A/B、绑定到当前事件池
5. 在 Event Config 区微调参数（volume/rate/detune/cooldown/maxVoices/strategy）
6. 先点 `Apply Runtime` 在线验证
7. 确认后点 `Apply + Save` 持久化到 `audio-profile.json`
8. 可通过 `audioManager.exportDiff()` 导出当前差异

## 6. 音频素材库接入

当前仓库默认 `audio-library.manifest.json` 为空（避免素材未就绪时产生无效加载）。

接入步骤：

1. 将素材放入目录（示例）：`assets/audio`
2. 生成清单：

```bash
python tools/generate_audio_manifest.py --input-dir assets/audio --base-path assets/audio --output audio-library.manifest.json --preload-limit 24
```

3. 在游戏内 Audio Debug 面板点击 `Reload Manifest`
4. 在 Library 区确认素材已可搜索、试听和绑定

说明：

- 若希望禁用启动预加载，可把 `preloadLimit` 设为 `0`
- 支持扩展名：`.ogg,.mp3,.wav,.m4a,.aac`

## 7. 新增音频事件规范

1. 在 `src/audioEventRegistry.js` 新增事件定义
2. 在业务代码锚点处触发：

```js
this.playAudioEvent?.('your_event_id', { optionalMeta: true });
```

3. 通过面板绑定素材并调参，最后 `Apply + Save`

命名建议：

- UI：`ui_*`
- 玩家行为：`player_*`
- 猎物行为：`prey_*`
- 拓扑编辑：`topology_*` / `edit_*`
- 进程节点：`stage_*` / `game_*`

## 8. 发布态与开发态

- 调试面板默认仅开发态启用（`CORE_DEMO_DEBUG !== false`）
- 发布态可将 `CORE_DEMO_DEBUG` 设为 `false` 关闭面板 UI
- 即使关闭面板，`CoreAudioManager` 与事件播放链路仍可运行
