# 猎物状态机与追逃机能说明

## 1. 这份文档解决什么问题

这次重构的目标不是单纯把 prey 跑快一点，而是把下面三件事一起做实：

1. 先量出玩家集群在当前架构下的真实追猎机能。
2. 再让 prey 的追逃状态机基于这份机能工作，而不是凭感觉配速度。
3. 在 richer AI 落地后，仍然把 `updatePrey` 的成本控制住。

用户侧的核心反馈很明确：

- 现在的猎物“会跑但不明显”。
- 逃跑更多像持续往外滑，不像有生态感的惊逃、散开、回稳和保命。
- 设计上应该让玩家短时间内有“要不要追、怎么追”的判断，而不是全程轻松平推。
- 但整体难度仍然要偏轻松休闲，保留后期滚雪球和扫荡爽感。

所以这次重构的原则是：

- 第一目标是生态感和动感。
- 第二目标是轻度决策和压力。
- 第三目标才是数值难度。

---

## 2. 旧实现的主要问题

旧版 `src/sceneEnemies.js` 的 prey 行为本质上还是一套连续 steering：

- 用 `danger` 驱动单帧 flee 向量。
- archetype 差异主要落在 `speed / accel / wander / weakAngle / guardPulse` 的乘法修饰。
- school 是局部 flocking，bulwark/weakspot/apex 是在 flee 上叠几个特殊分量。
- 没有离散状态，也没有短时爆发、回稳、记忆、群体告警。
- 速度上限完全看 prey 自身静态 `speed`，没有参考玩家当前真实机能。

这会导致几个直接问题：

1. 猎物缺乏“状态感”。玩家看不出它现在是巡游、警觉、爆发逃逸还是回稳。
2. 追逃节奏太平。没有明显的窗口期，自然也就没有“差一点追上”和“被它甩开一点”的体验。
3. 同一个 prey 数值在小体量和大体量玩家面前都可能不合理。
4. 邻域查询是全表扫，后面如果把状态机做复杂，`updatePrey` 的成本会上涨得很快。

---

## 3. 新实现概览

本次改动新增了三层。

### 3.1 机能测量层

新增 `ecoTelemetry`，按：

- 集群规模桶：`small / medium / large`
- 驱动相位：`stable / cruise / pursuit / hunt / burst`

持续记录：

- 玩家质心速度
- 节点平均速度
- 集群跨度
- 压缩 / 舒张状态
- prey 状态驻留时间
- prey chase 的启动、脱离、被吞和爆发次数

运行时可以通过：

- `window.dumpEcoTelemetry()`
- `showTelemetryOverlay`

直接查看当前追逃埋点。

### 3.2 新状态机层

prey 现在不再只是“危险越高越往外逃”，而是进入明确状态：

- `graze`
- `schooling`
- `alert`
- `evade`
- `burst`
- `recover`
- `brace`

这些状态由以下信息共同驱动：

- 当前驱动相位
- 玩家质心闭距速度
- 附近节点压迫
- 群体告警
- 自身体力 `stamina`
- 目标弱点 / 防御需求

### 3.3 性能层

新增通用空间哈希：

- `buildSpatialHash`
- `querySpatialHash`

并让 prey 邻域查询改走缓存：

- `pickNearbyNodes`
- `pickNearbyPrey`
- `pickNearestNode`

这样 richer prey AI 不会继续把邻域查询成本线性放大。

---

## 4. 追猎机能测量方法

新增脚本：

```bash
node tools/prey_mobility_report.js
```

它会在无渲染 Node 环境里复用当前移动核心，按不同节点规模和驱动相位跑出稳定数据。

注意：

- 单位是当前世界空间单位 / 秒，不是屏幕像素。
- 这里更重要的是相对关系，不是绝对数值。
- prey 状态机实际也是按“玩家能力比例”在设计，而不是死盯某个绝对速度。

## 5. 当前机能基线

下面这张表是最值得直接给程序和 AI 开发看的摘要。它只保留三个最关键的玩家体量档和三个最关键的姿态档。

| 节点数 | 姿态 | 质心均速 | 平均跨度 |
| --- | --- | ---: | ---: |
| 6 | `stable` | 467.7 | 185.3 |
| 6 | `cruise` | 822.6 | 268.1 |
| 6 | `burst` | 2588.5 | 617.9 |
| 13 | `stable` | 829.1 | 390.2 |
| 13 | `cruise` | 1607.0 | 671.0 |
| 13 | `burst` | 3954.1 | 1343.4 |
| 22 | `stable` | 1358.9 | 837.4 |
| 22 | `cruise` | 2693.1 | 1427.0 |
| 22 | `burst` | 7113.5 | 2899.8 |

可直接读出的结论：

1. 玩家在 `burst` 态下的质心机能远高于 `cruise`，所以 prey 想制造“差一点跑掉”的体验，必须依赖短窗口爆发，而不是常驻提速。
2. 玩家规模越大，质心机能和覆盖跨度增长都很明显，所以后期 prey 的相对能力要主动下修，保证成长爽感。
3. `stable` 到 `cruise` 的差距已经足够大，说明“收缩 / 巡航 / 前扑”本身就是设计资源，prey 应该围绕这个差做文章。

## 6. 新状态机怎么工作

### 6.1 追逃不是看 prey 自己的静态速度，而是看玩家当前能力参考值

新逻辑里，prey 会先拿到一份玩家当前能力参考：

- 当前规模桶
- 当前驱动相位
- 该桶该相位的平均质心速度

然后用状态机里的比例系数去求目标速度：

- 巡游态：只取玩家能力的一小部分
- 警觉态：明显抬高
- 逃逸态：接近玩家巡航能力
- 爆发态：短时间接近或略高于当前玩家能力
- 回稳态：重新掉回去

这就是为什么新 prey 会更像“有命保的活物”，而不是一条持续匀速飘走的点。

### 6.2 状态含义

- `graze` / `schooling`
  - 低压巡游态。
  - 保留 wander、群聚和低速生态感。

- `alert`
  - 第一层警觉。
  - 开始朝逃逸向量转身，但不会立刻把速度拉满。

- `evade`
  - 持续逃逸态。
  - 主要承担“让我先拉开一点点距离”的体验。

- `burst`
  - 短时爆发逃逸。
  - 是追逃里的关键窗口，不常驻。

- `recover`
  - 爆发后回稳。
  - 给玩家一个可追上的反扑窗口。

- `brace`
  - 仅给 `bulwark / apex` 的抗压状态。
  - 充能、顶住、反推，不靠单纯加速。

### 6.3 状态切换依据

不是只看距离，而是综合：

- 玩家驱动相位带来的侵略度
- 玩家质心闭距速度
- 附近节点压迫强度
- prey 是否已经被挂住
- 群体告警
- 自身体力
- `compressionAccess`

## 7. 各 archetype 的设计定位

### `skittish`

- 最敏感。
- 允许最明显的 `burst`。
- 目的是制造“差一点就追到”的轻度追击张力。

### `school`

- 常态更重群体行为。
- 一旦一只告警，会把整群拉进 `alert / evade / burst`。
- 目的是制造生态里“先收拢，再炸散”的视觉和节奏。

### `bulwark`

- 不是跑得最快，而是会 `brace`。
- 通过守卫脉冲、顶住和反推告诉玩家：不是一直铺开就最优。

### `weakspot`

- 更强调 `alert / evade` 时的护弱点和横移。
- 让玩家必须考虑角度和包围。

### `apex`

- 混合 `brace + weakspot protection + delayed burst`。
- 它不是纯 boss，而是终局生态目标。

## 8. 以后调 prey，优先改哪里

如果要继续调状态机，优先看这几个量，而不是先改血量：

1. `capabilityRatios`
   - prey 对玩家当前机能的相对速度比例。

2. `burstDuration`
   - 决定“甩开一下”的时间有多长。

3. `recoverDuration`
   - 决定玩家爆发后能不能立刻追回来。

4. `braceEnter / braceRelease`
   - 决定 `bulwark / apex` 多早开始用保命能力。

5. `dominanceDrop`
   - 决定玩家越大时，prey 相对能力掉得有多快。

推荐理解方式：

- 想让前期更有追逐感：先加 `alert / evade / burst` 的能力比例。
- 想让后期更爽：先加 `dominanceDrop`，不要盲目削 prey 基础速度。
- 想让某类 prey 更有“保命手段”：先改状态切换门槛，不要直接加血。

## 9. 性能改动

这次和状态机一起落的性能优化有两点。

### 9.1 邻域查询缓存

`updatePrey` 每帧先建立：

- node spatial hash
- prey spatial hash
- group cache

之后：

- `pickNearbyNodes`
- `pickNearbyPrey`
- `pickNearestNode`

都不再默认扫全表。

### 9.2 telemetry 不存全量历史

埋点只存：

- rolling current
- cumulative averages
- 少量 recent chases

不存大数组逐帧历史，避免开发期埋点本身拖慢游戏。

---

## 10. 对后续开发的直接建议

1. 不要再回到“prey 只靠一条 flee 向量”的写法。
2. 新增 prey 时，先定义它逼玩家进入哪个姿态，再决定它的 state profile。
3. 如果要判断 prey 是否过强/过弱，先跑 `node tools/prey_mobility_report.js`，再看 `window.dumpEcoTelemetry()`。
4. 如果体验上“还是太好追”，优先拉高 `alert -> evade -> burst` 的状态窗口，而不是无脑提高常驻速度。
5. 如果体验上“太折磨”，优先缩短 `burstDuration`、加快 `recover`，让玩家更快拿到追上窗口。

---

## 11. 这次重构的实际效果

相对旧版，这版 prey 的核心变化是：

- 逃跑更明显了，因为它们不再常驻匀速，而是有真正的状态节奏。
- 生态感更强了，因为 school 会传警、bulwark 会顶、weakspot 会护角度。
- 追逐更有判断了，因为玩家需要决定现在是继续追、先压缩、还是等它爆发后回稳再吃。
- 成长爽感被保住了，因为 prey 相对能力会随着玩家规模上升而下修。

这才是当前项目里“轻松但不无聊”的正确方向。
